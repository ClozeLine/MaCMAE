# Project: MaCMAE

Self-supervised MAE pretraining on Mars crater imagery + a frozen-encoder
linear probe for crater degradation state. See `README.md` for the full
description, data source, and run commands.

**Research question:** Does self-supervised (MAE) pretraining on Mars crater
imagery learn visual representations that capture crater degradation? Labels
(A/B/C) are used only at probe time — pretraining is label-free.

## Layout

```
thesis/
├── config.py                  # paths + hyperparameters (env-overridable)
├── dataset.py                 # dataset classes + group-aware train/val/test split
├── pretrain/
│   ├── mae.py                 # HF ViT-MAE wrapper
│   └── train_mae.py           # Phase 1: MAE pretraining (resumable)
└── probe/
    ├── extract_embeddings.py  # encoder → embedding cache (.parquet)
    └── linear_probe.py        # Phase 2: logistic regression on frozen embeddings
```

## Conventions

- One venv managed with **uv** (`uv sync`). Use `uv`, not `pip`.
- Run everything as a module from the repo root:
  `uv run python -m thesis.pretrain.train_mae`,
  `uv run python -m thesis.probe.linear_probe --embeddings ...`.
  Paths in `thesis/config.py` are `__file__`-anchored; module-style is the
  documented invocation.
- New files go inside the relevant subpackage (`pretrain/` or `probe/`).

## Data & outputs (gitignored)

- `thesis/data/` — crater patches + `metadata.csv`, published on Zenodo (not in
  the repo). `metadata.csv` `image_path` is relative to `thesis/data/`.
- `thesis/{checkpoints,embeddings,results,figures}/` — regenerated by running
  the pipeline.
- `_archive/` — local-only out-of-scope code (dataset builders, analysis/graph
  scripts, cluster jobs) kept out of the published repo.
